## Код

Перед імплементацією перевір - чи вже є готове рішення, що можна перевикористати. Мінімізуй кількість коду.

Розбивай скрипти на окремі файли. Не створюй god-object'и. Дотримуйся low coupling.

## Тестування

По закінченню завдання - створи unit тести для перевірки нового ф-ціоналу. Або відредагуй вже наявні.

Перевір, чи ф-ціонал працює відповідно очікуванням. Якщо ні - опиши проблему і очікуй інструкцій.

## Коміти

Коли поступає запит на коміт - спочатку необхідно переглянути зміни та стиль оформлення попередніх комітів.

Перед тим, як закомітити зміни - треба внести правки в документацію, якщо вони необхідні.

- main: README.md;
- backend: GalleryApp\backend\README.md;
- frontend: GalleryApp\frontend\README.md.

## Оформлення комітів

```
<marker>: <description>
```

| Маркер     | Значення                                                                                            | Приклад                                         |
| ---------- | --------------------------------------------------------------------------------------------------- | ----------------------------------------------- |
| `feat`     | Додавання нової функціональності                                                                    | `feat: add player inventory`                    |
| `fix`      | Виправлення помилок (багів)                                                                         | `fix: prevent crash on load`                    |
| `chore`    | Технічні зміни (оновлення залежностей, інструментів, конфігурацій) без прямого впливу на функціонал | `chore: update Unity version to 2023.3`         |
| `refactor` | Переписування або реструктуризація коду без зміни поведінки                                         | `refactor: improve movement script readability` |
| `style`    | Зміни, що не впливають на код (форматування, відступи, крапки з комами тощо)                        | `style: format Player.cs script`                |
| `docs`     | Документування, коментарі                                                                           | `docs: update README with instructions`         |
| `test`     | Додавання або оновлення тестів                                                                      | `test: add unit tests for EnemySpawner`         |
| `perf`     | Покращення продуктивності                                                                           | `perf: optimize collision checks`               |
| `ci`       | Зміни у конфігурації CI/CD                                                                          | `ci: add GitHub Actions workflow`               |
| `build`    | Зміни, що впливають на систему збірки                                                               | `build: update WebGL build settings`            |
| `revert`   | Відкат до попереднього стану                                                                        | `revert: feat: add jumping ability`             |

- Коміт починається з маркера, після якого ставиться двокрапка (`:`);
- Опис коміту пишеться в імперативному стилі (як наказ, без минулого часу);
- Перше слово після маркера — з маленької літери (окрім імен власних чи класів);
- Можна додати область, яку зачіпає зміна, у круглих дужках.

```
feat(player): add double jump
fix(ui): correct button alignment
```
